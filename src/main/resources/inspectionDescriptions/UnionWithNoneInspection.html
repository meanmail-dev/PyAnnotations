Using Optional[...] instead of Union[..., None] is preferred in Python type annotations because:

  • It more clearly communicates the intent that a value might be None
  • It's more concise and easier to read
  • It follows the standard pattern recommended in the typing module documentation

This inspection helps maintain consistent and idiomatic type annotations in your code.
