This inspection detects TypeVar used in function annotations without meaningful binding.
def foo(x: T) -> int — T is used only once, doesn't constrain anything
def bar(x: T) -> T — T is used meaningfully (input/output relationship)

A TypeVar should be used at least twice in a function signature to establish a type relationship (e.g., between parameters, or between a parameter and return type). Using a TypeVar only once is equivalent to using Any and provides no additional type safety.

Note: This inspection uses heuristics to detect TypeVars (single uppercase letters, names ending with _T, etc.) and may have false positives.
